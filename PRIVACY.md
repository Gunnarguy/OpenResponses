# Privacy Policy

Last updated: 2026-05-29

OpenResponses is a native iOS client for the OpenAI Responses API, built with a local-first design philosophy. We believe that your data belongs on your device. This Privacy Policy details what information is collected, what data stays local, what data is transmitted over the network, and how you retain complete control over your credentials and chat histories.

---

## 1. Plain-Language Privacy Summary

- **Zero Middleware Servers:** OpenResponses does not route your data through any intermediate servers or proxy networks. All API requests are made directly from your device to the configured API provider (e.g. OpenAI or Notion workspace).
- **No Third-Party Analytics SDKs:** The application does not integrate tracking frameworks (such as Firebase, Mixpanel, Amplitude, or Google Analytics). Your usage behavior is private and stays on your device.
- **You Own Your Keys:** Your API keys and integration tokens are stored locally on your device in the secure iOS Keychain. We have no way to access or collect them.

---

## 2. On-Device Storage (What Stays Local)

The following information is persisted within the app’s local sandboxed directory or Keychain enclave:
1. **API Keys & Credentials:** The application saves OpenAI keys, Notion access tokens, and custom MCP authentication headers inside the Secure Enclave-backed iOS Keychain.
2. **Conversation History:** Saved prompts, active chat threads, model settings presets, and execution statistics reside in the local directory as JSON files via `ConversationStorageService`. No data is uploaded to iCloud or other backup networks unless you explicitly opt in to the system-managed backup.
3. **Diagnostic Traces:** Logging messages generated by `AppLogger` are kept in local memory buffers. They are displayed in the in-app log viewer and are never exported unless you manually trigger a log export.
4. **Security-Scoped Bookmarks:** When you import files or folders for processing, Apple generates security-scoped bookmarks stored on-device to retain access permissions across launches.

---

## 3. Data Sent Off-Device (Network API Boundaries)

To execute completions and run tools, OpenResponses transmits data over HTTPS directly to the following external APIs:

| Recipient Service | Data Transmitted | Purpose of Transmission | User Opt-Out / Controls |
| :--- | :--- | :--- | :--- |
| **OpenAI Responses API** | User messages, system instructions, temperature/nucleus sampling parameters, tool definitions (web search, code interpreter, computer use), and session metadata. | Process input prompts and stream response completion deltas. | First request is gated behind an explicit permission notice. Settings can be configured to toggle tool integrations. |
| **OpenAI Embeddings API** | Plaintext document segments or user search query phrases. | Generate semantic vector embeddings for local-first document indexing. | Can be disabled by turning off the File Search tool in settings. |
| **Notion API** *(Optional)* | Notion integration access tokens, database schemas, and page content parameters. | Enable the assistant to search databases and write records. | Disabled by default. Can be disconnected or revoked in settings at any time. |
| **Local Network Bridge** *(Optional)* | Viewport screenshots, mouse coordinate pairs, click parameters, key commands, and terminal lines. | Automate actions via the local `computer` or `computer_use_preview` tools. | Gated by a local network permission prompt and an explicit step-by-step UI approval dialog. |

---

## 4. Document & File Uploads

When you attach files (PDF, TXT, PNG, etc.) to the chat:
- The app uses `FileConverterService` to extract plaintext or render images locally.
- Extracted content and image buffers are sent as part of the API payload request to the OpenAI Responses API.
- Files are not uploaded to any other server or repository.

---

## 5. App Store Privacy Declarations

For App Store compliance, the application declares the following data usages:
- **Contact Info / Identifiers:** Not Collected.
- **Diagnostics:** The only diagnostic data processed are system crash logs managed natively by Apple's opt-in system reporting. These reports contain stack traces and do not include API keys, chat text, or user files.
- **User Content:** Processed solely to generate completions via direct connections to OpenAI. No user content is collected by the developer.

---

## 6. How Users Can Delete and Reset Data

You have absolute control over your local and remote storage traces. Under **Settings → General**, tapping **Reset All Settings** will:
1. Purge all Keychain generic password entries associated with OpenResponses.
2. Erase the on-device conversation database and prompt libraries.
3. Revoke all local folder and file security-scoped bookmark references.
4. Return the application to the initial onboarding welcome flow.
5. In addition, you can toggle **Store Responses** to `false` in settings to instruct OpenAI's servers to execute requests in a stateless manner.
