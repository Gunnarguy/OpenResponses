⚡ Optimize DateFormatter initialization in BatchJobsView

💡 **What:** Replaced the local `DateFormatter` initialization inside the `formatTimestamp` method with a static cached instance (`Self.dateFormatter`) in `BatchJobsView.swift`.
🎯 **Why:** `DateFormatter` initialization is notoriously expensive in Swift. Since this method is called frequently (e.g. for every list item rendered on screen), instantiating a new formatter on every render pass is a CPU and memory bottleneck. Caching the thread-safe formatter provides a significant performance improvement.
📊 **Measured Improvement:** Since `swift` and `xcodebuild` are not available in this environment, it was impractical to run a standalone benchmark test. However, based on Apple's documentation and known performance characteristics of Foundation, reusing a `DateFormatter` instance commonly yields a 10x-50x speedup for formatting operations compared to repeated initializations, drastically reducing allocations and overhead on the UI thread.
